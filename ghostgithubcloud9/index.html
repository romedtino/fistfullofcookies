Ghost for GitHub Pages Using Cloud 9

So you want to start a blog?

Yes. Yes I do. Although, I had no idea where to start (even though there's thousands of resources on Google). I know I wanted free hosting because I am beyond cheap. I know I wanted the website to at least have SSL because you owe it to yourself to have some level of dignity security. Hey, cool! GitHub Pages has exactly what I am looking for! Now what blog service do I turn to?

I mean you probably have heard of a plethora of different blog services such as Xanga or LiveJournal just to name a few of the more popular ones ;). However, the ones that caught my eye when I was perusing the depths of the internet were Tumblr, WordPress, Blogger, Jekyll and Ghost. If you can't tell, I did opt for Ghost for these reasons:

Tumblr - There's just too much going on here and I can't take it seriously. I understand
the reasons why certain people choose Tumblr (questionable if that was Tumblr's initial intention) and it's really more of getting away from the blog mentality with just sharing images, a quick prose or something that doesn't necessarily need to be chunky. In this day and age where people want their information quick and gritty, Tumblr has your back. Also for my use case, certain proxies I have played with for some reason like to block Tumblr so that's just a no go.

WordPress - WordPress by WordPress**.org** is actually pretty nice but it seems really heavy and cumbersome. I wanted something extremely simple (even though I have probably convoluted my process) but also customizable.

Blogger - You have to tie this to a Google account and even though I almost have everything tied to Google, I wanted to go a different route. Blogger is also pretty different where in that you don't have the ability to host the server on your own.

Jekyll - You can host on your own, it is open source, it is meant for actually generating blogs or static pages, so why did I not choose Jekyll? Honestly, it's because I tried Ghost first and haven't had the chance to try Jekyll. Also, I have used Node.js before and not really Ruby so I didn't want to learn something on top of another.

Ghost - Looks cool, decent documentation and examples to get started, and it doesn't seem too hard to set up. Let's do it!

Creating your blog

GitHub Pages

GitHub Pages is awesome. It's a place where you can host static data for either a project or in our case, a blog!

  1. Create a GitHub Pages site using these directions.
  • Note: Try to use your main account for your blog or prepare to be savvy and do some cleanup before posting your blog (e.g. if your GitHub username is benfranklin, host with your main github pages: benfranklin.github.io as opposed to benfranklin.github.io/bensblogaboutbells)
  1. Leave it blank for now and there's no need to apply any theme since you're most likely going to find your own Ghost theme.

C9.io

Cloud9 is also awesome. It's literally a free online linux workspace you can almost do anything with. I have used it to debug/create my personal website as well as test my Chromecast app - Kings Cup Cast. Both so far have been javascript but you can do A LOT more from C++, Java, Python, etc. The only scary thing here is that they initially ask you for a credit card despite having a free version so watch out for that.

  1. https://c9.io/
  2. Create a new workspace BUT don't tie it to your GitHub account just yet.
  3. C9.io uses Node.js! but by default it uses v4 while you'll need at least 6. So go ahead and get those and set it as default
  4. nvm install 6
  5. nvm alias default 6

Ghost

  1. You can cheat and just install it via npm install ghost BUT for some reason I went the hard way and compiled it from source.
  2. Visit https://ghost.org/developers/ and grab the zip. By the time of my writing it was wget https://github.com/TryGhost/Ghost/releases/download/1.0.2/Ghost-1.0.2.zip
  3. unzip Ghost*.zip
  4. npm install --production
  5. If you run into any conflicts (I know I did, feel free to just npm uninstall that dependency and use the one Ghost needs. If you need that specific version, then why are you using the same c9 workspace?! :O Make a new one!) You might also need apt get python-devel.
  6. Once ghost is installed, you would have to edit ghost/config.js for cloud 9 specificities that deal with "localhost" and ghosts default URLs.
  7. Change http//url: 'http://localhost:2368' to url: 'https://' + process.env.C9_HOSTNAME
  8. Change host: '127.0.0.1' to host: process.env.IP
  9. Change port: 2368 to port: process.env.PORT

Now you're all set!

Running Ghost

Once ghost is up and running, simply do npm start and hopefully ghost starts running! Just simply visit your https://WORKSPACE-USERNAME.c9users.io and you should see a Ghost Splash screen! Then just visit https://WORKSPACE-USERNAME.c9users.io/ghost to login.

Phew!

Publishing to GitHub

So you've gotten your blog all nice and nifty, now you want to publish it to your GitHub page. What's nice about c9 is that it also comes with Python pre-installed!

  1. Install this nifty tool someone made that transcribes Ghost blogs into static pages.
  2. pip install buster
  3. Now you're ready to summon the GitHub page you had previously created. I decided to put mine within the ghost installation as this makes the next few steps easier (less command line arguments)
  4. git clone git@github.com:username/username.github.io.git static
  5. Now let buster generate the static pages
  6. buster generate --domain=https://WORKSPACE-USERNAME.c9users.io
  7. Then deploy!
  8. buster deploy

If things worked out correctly, it should have checked it all in your GitHub page! Yay!

Addressing random problems

Problem: You decided to use a project page as opposed to your main page on GitHub to host your blog.

I had this exact problem. buster unfortunately, is not perfect. Currently my /content directory assumes it sits in the root but this isn't always the case (e.g. http://username.github.io/myawesomeblog/content)

My solution is pretty hacky for my taste but for now, it works.

sed -i 's:="/content:="/myawseomeblog/content:' ./static/index.html

Yea, it's a sed command and I kind of don't like it since I am post processing the output which makes me question if my approach was worth it. I did wrap most of these in my bash_aliases to save more time. Nevertheless, it all seems to work! :D

Thanks for reading!
